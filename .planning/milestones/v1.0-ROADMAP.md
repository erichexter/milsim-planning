# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-03-17
**Phases:** 1–4
**Total Plans:** 20

## Overview

Full MVP of the RP0 Airsoft Event Planning Platform. Covers secure authentication and
RBAC, full commander workflow (events, roster import, hierarchy), content and notification
pipeline, and the player-facing mobile experience with roster change requests. At the end
of this milestone a commander can run an entire event briefing from one URL and every
player can find their assignment on a mobile phone.

## Phases

### Phase 1: Foundation

**Goal**: Every user can securely access the application and all data is scoped correctly by role
**Depends on**: Nothing (first phase)
**Plans**: 4 plans

Plans:
- [x] 01-01: Solution scaffold, EF Core entity model + migration, authorization policy registration
- [x] 01-02: Test project stubs + auth endpoints (login, magic link, password reset, logout, invite) + JWT service
- [x] 01-03: RBAC handlers + IDOR scope guard: MinimumRoleHandler, ScopeGuard, CurrentUserService, AUTHZ integration tests
- [x] 01-04: React SPA foundation: Vite scaffold, auth.ts, api.ts, useAuth hook, ProtectedRoute, all 5 auth pages

**Details:**
Established ASP.NET Core Identity with JWT bearer auth. Implemented MinimumRoleHandler as single
source of truth for role hierarchy. Magic link GET returns HTML form, POST completes auth (email
scanner protection). ScopeGuard.AssertEventAccess is first line of every service method.

### Phase 2: Commander Workflow

**Goal**: A Faction Commander can create an event, import their player roster via CSV, and organize players into platoons and squads
**Depends on**: Phase 1
**Plans**: 5 plans (originally 6 in roadmap, plan 02-06 was merged into 02-05)

Plans:
- [x] 02-01: Phase 2 entity model (Event/Faction/Platoon/Squad/EventPlayer) + EF migration + test stubs
- [x] 02-02: Event CRUD API (create, list, publish, duplicate) + integration tests
- [x] 02-03: CSV roster import API (validate + commit endpoints, CsvHelper pipeline, invite trigger)
- [x] 02-04: React UI: EventList, CreateEventDialog, DuplicateEventDialog, EventDetail, CsvImportPage
- [x] 02-05: Hierarchy API (platoon/squad CRUD, player assignment, roster tree) + React HierarchyBuilder + RosterView

**Details:**
Delta EF migration over drop-and-recreate. Two-phase CSV import (validate → preview → commit).
Bulk assign with `squad:{guid}` / `platoon:{guid}` destination encoding. Profile callsign
overrides roster CSV callsign (precedence rule in PlayerController and HierarchyService).

## Milestone Summary

**Key Decisions:**

- MinimumRoleHandler is sole source of role hierarchy evaluation — no raw role strings in business logic
- ScopeGuard.AssertEventAccess as first line of every service method (IDOR prevention contract)
- Delta EF migrations (never drop-and-recreate) — keeps InitialSchema intact
- Two-phase CSV import (validate → preview → commit) — row-level error reporting before any save
- Profile callsign (UserProfile.Callsign) overrides roster callsign (EventPlayer.Callsign)
- Bulk assign destination encoding: `squad:{guid}` / `platoon:{guid}`
- Pre-signed URLs generated on demand from R2Key, never persisted
- Notification blast async via Channel + BackgroundService; 202 returned after enqueue
- IntegrationTestAuthHandler centralizes test auth across all suites
- AWS SDK v4 for Cloudflare R2; ForcePathStyle required

**Known Gaps (proceeded without full requirement closure):**

- RCHG-01..05 and PLAY-01..06 checkboxes in REQUIREMENTS.md were not updated to reflect
  Phase 4 completion. These features were shipped in Phase 4 but noted as "Pending" in the
  traceability table. Archived as-is per user decision.

**Technical Debt:**

- HierarchyBuilder bulk-assign refactor removed old drag-and-drop — no dnd-kit dependency needed
- Test suite had 4 stub test files deleted; new HierarchyBuilder tests cover new component structure
- DevSeedService BackfillCommanderMembershipsAsync required null guard for events with Faction=null
