## Architecture

**Pattern:** Layered MVC — single backend project with Controllers → Services → DbContext. Not Clean Architecture (no separate Core/Infrastructure projects). All business logic lives in `Services/`.

```
milsim-planning/
├── milsim-platform/src/MilsimPlanning.Api/      ← .NET 10 ASP.NET Core Web API
│   ├── Controllers/                              ← HTTP surface; thin, delegate to services
│   ├── Services/                                 ← All business logic
│   ├── Data/
│   │   ├── Entities/                             ← EF Core entity classes
│   │   ├── Migrations/                           ← EF delta migrations
│   │   ├── AppDbContext.cs                       ← Single DbContext (IdentityDbContext<AppUser>)
│   │   ├── DevSeedService.cs                     ← Dev seed (commander@dev.local, player@dev.local)
│   │   └── ProductionSeedService.cs              ← Production bootstrap (roles + admin account)
│   ├── Models/                                   ← DTOs ({Feature}Dto) and request models ({Action}{Feature}Request)
│   ├── Domain/                                   ← AppRoles — role hierarchy constants
│   ├── Authorization/                            ← ScopeGuard (IDOR prevention), MinimumRoleHandler
│   └── Infrastructure/BackgroundJobs/            ← Email notification queue + worker
├── milsim-platform/src/MilsimPlanning.Api.Tests/ ← Integration tests (Testcontainers + WebApplicationFactory)
└── web/                                          ← React 19 + Vite + TypeScript frontend
    ├── src/pages/                                ← Page components (route-level)
    ├── src/components/                           ← Feature + shared components
    ├── src/lib/                                  ← api.ts, auth.ts, utils.ts
    ├── src/hooks/                                ← useAuth, useTheme
    └── src/__tests__/ + src/tests/              ← Vitest + Testing Library frontend tests
```

**Domain model:** One `Event` → one `Faction` → many `Platoon`s → many `Squad`s → many `EventPlayer`s. Each event has an `EventMembership` table linking `AppUser` to event with a role.

## Coding Conventions

See `CODING-STANDARDS.md` for the authoritative reference. Summary:

### Naming
- **Files (backend):** PascalCase (`HierarchyService.cs`, `EventDto.cs`)
- **Files (frontend):** PascalCase for components (`HierarchyBuilder.tsx`), camelCase for hooks (`useAuth.ts`) and lib (`api.ts`)
- **DTOs:** `{Feature}Dto` for responses, `{Action}{Feature}Request` for requests
- **Tests (backend):** `{Feature}Tests` class, `{Feature}_{Scenario}_{ExpectedResult}` method

### Structure
- Controllers are thin — validate input, call service, return result
- Services contain all business logic — throw exceptions, controllers catch and map to HTTP
- Average file: 50–200 lines; largest non-migration file is `HierarchyBuilder.tsx` (746 lines) — SRP concern flagged

### Error handling (backend)
```csharp
// Services throw; controllers catch:
KeyNotFoundException  → 404
ArgumentException     → 400
ForbiddenException    → 403 (+ caught by global middleware in Program.cs)
InvalidOperationException → 409
```
All error responses use ASP.NET Core `ProblemDetails` shape:
```json
{ "type": "...", "title": "Forbidden", "status": 403, "detail": "..." }
```

### Testing pattern (backend)
```csharp
// Typical test structure:
public class HierarchyTests : HierarchyTestsBase  // extends base with factory + clients
{
    [Fact]
    public async Task CreatePlatoon_AsCommander_ReturnsCreatedPlatoon()
    {
        // Uses WebApplicationFactory + IntegrationTestAuthHandler
        // Real PostgreSQL via Testcontainers (PostgreSqlFixture)
        // No mocked DB — all tests hit real DB
    }
}
```

### Frontend patterns
```typescript
// All API calls through lib/api.ts
// Server state via TanStack Query (@tanstack/react-query)
// Forms via react-hook-form + zod
// Styling: Tailwind CSS v4 + shadcn/ui components
// No direct fetch() in components
```

## Type Conventions

- **Primary key type:** `Guid` in C#
- **Route ID constraints:** `{id:guid}` — e.g., `[HttpGet("{eventId:guid}")]`
- **TypeScript ID representation:** `string` (UUID format: `"xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"`)
- **Nullable fields (C#):** `string?` — e.g., `public string? Callsign { get; set; }`
- **Nullable fields (TypeScript):** `string | null` — e.g., `callsign: string | null`
- **Exception:** `Faction.CommanderId` is `string` (FK to ASP.NET Identity `AppUser.Id`, which is a string GUID)
- **Example Guid from live data:** `01802cfe-fc30-43d2-ac41-f552059000a78` (commander user ID from dev seed)
